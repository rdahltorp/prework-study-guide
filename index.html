Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • always check to make sure your branch is running off the most recent code by using the following command: git pull origin [name of main branch]
  • To get ready to push your latest feature to GitHub and merge with the main branch:
    1. Confirm you are in the proper working branch that you have been updating.
    2. Stage your changes by entering command: git add -A. This tells git to expect that all the items staged will be sent in a pull request.
    3. Now, to commit the staged changes to the repo, use the command: git commit -m [INSERT A SMALL MESSAGE ABOUT THE COMMIT]. The -m flag notes an associated message.
    4. Confirm the branch code you are working off of is the most upto date with command: git pull origin [NAME OF YOUR MAIN BRANCH]
    5. If display says you are up to date, it is time to push with command: git push origin [WORKING BRANCH NAME]
    6. Then your pull request should be in GitHub!
